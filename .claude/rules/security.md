# セキュリティルール

## 絶対禁止
- シークレット・APIキー・パスワードをコードにハードコードしない
- `.env` ファイルをコミットしない
- `eval()` や動的コード実行を使わない

## 入力バリデーション
- ユーザー入力は必ずバリデーションする（システム境界のみ）
- SQLクエリはプレースホルダーを使う（文字列連結禁止）
- コマンド実行時は引数をエスケープする

## 依存関係
- `npm audit` を定期実行する
- 不要な依存パッケージは追加しない
- パッケージは用途を確認してからインストールする

## 環境変数
- シークレットは環境変数経由で渡す
- `.env.example` にキー名だけ記載し、値は書かない
