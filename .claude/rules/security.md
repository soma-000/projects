# セキュリティ・運用ルール

## 認証情報の管理

- X API Key / Secret、Bearer Token は絶対にコードにハードコードしない
- GitHub PAT も同様にコミット禁止
- 認証情報は環境変数か `CLAUDE.local.md` に記載する
- `.env` ファイルはコミットしない（.gitignore で除外済み）

## X API 利用上の注意

- レート制限を必ず確認してからスクリプトを実行する
- 大量リクエストはバッチ処理 + sleep で分散させる
- 本番実行前にドライランで動作確認する
- X規約に違反する自動化（スパム、水増しなど）は行わない

## スクリプト実行前チェック

- 意図しないデータ送信・投稿が起きないか確認する
- 本番APIとテスト環境を混同しない
- スケジュール実行は初回必ず手動で動作確認してから有効化する
