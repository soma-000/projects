# コーディング規約

## 命名規則
- 変数・関数: `camelCase`
- クラス・型: `PascalCase`
- 定数: `UPPER_SNAKE_CASE`
- ファイル名: `kebab-case`

## 関数の原則
- 1関数1責務
- 引数は3つ以内。超える場合はオブジェクトにまとめる
- 副作用のある関数は名前で明示（`fetchUser`, `saveFile` など）

## コメント
- 「何をしているか」ではなく「なぜそうしているか」を書く
- 自明なコードにコメント不要
- TODO は `// TODO(名前): 内容` 形式

## テスト
- ビジネスロジックは必ずユニットテストを書く
- テスト名は `it('should ...')` 形式
- カバレッジ目標: 80%以上
- テストは実装と同時に書く（後回し禁止）

## Git
- ブランチ命名: `feat/`, `fix/`, `chore/`, `docs/` プレフィックス
- コミットメッセージ: Conventional Commits形式
- 1PR = 1機能 or 1バグ修正
